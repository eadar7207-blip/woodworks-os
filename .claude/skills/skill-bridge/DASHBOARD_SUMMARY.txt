================================================================================
TECHNICAL DETAILS
================================================================================

POLLING SYSTEM:
  - Interval: 2 seconds (configurable)
  - What updates: Skills, History, Stats
  - Retries: 3 attempts with 1s delay
  - Smart pausing when not needed

STATE MANAGEMENT:
  - AppState object tracks all state
  - executions Map for active tasks
  - history array for completed tasks
  - filters object for search/status
  - stats cache for performance metrics

API INTEGRATION:
  - GET /available-skills → List all skills
  - POST /invoke/{skill} → Execute synchronous
  - POST /invoke/{skill}/async → Execute async
  - GET /status/{id} → Check execution status
  - GET /history → Get execution history
  - GET /health → Health check

ERROR RECOVERY:
  - Network errors: Auto-retry 3x with 1s delay
  - API errors: Show error state + retry button
  - Invalid JSON: Fall back to string display
  - Timeout: After 30s show retry option

PERFORMANCE:
  - DOM caching (no repeated selectors)
  - Debounced search (300ms)
  - Event delegation (single listener for dynamic content)
  - Efficient re-rendering (only changed elements)
  - CSS transitions (200ms)

ACCESSIBILITY:
  - Semantic HTML structure
  - Form labels on all inputs
  - WCAG color contrast
  - Keyboard navigation support
  - Screen reader friendly
  - Focus indicators

HOW TO USE
================================================================================

1. INSTALL DEPENDENCIES:
   pip install flask flask-cors

2. START SERVER:
   python skill_bridge.py
   
3. OPEN DASHBOARD:
   http://localhost:9000

4. EXECUTE A SKILL:
   - Go to "Workflows" tab
   - Click "Execute" on a skill
   - Fill in parameters
   - Click "Execute" or check "Run asynchronously"
   - View results in modal

5. MONITOR EXECUTION:
   - Go to "History" tab
   - See all past executions
   - Search or filter by status
   - Click "Monitor" on running tasks
   - Progress bar updates every 2 seconds

6. VIEW STATISTICS:
   - Go to "Statistics" tab
   - See success rate, avg duration
   - View charts of status distribution
   - See top 5 most-used skills

================================================================================
TROUBLESHOOTING
================================================================================

Dashboard not loading?
  → Check Flask is running: curl http://localhost:9000/health
  → Check files exist: ls -lh static/app.js templates/index.html
  → Check Flask logs: tail -f skill_bridge.log

No data showing?
  → Verify API is returning data: curl http://localhost:9000/available-skills
  → Check browser console (F12): Look for fetch errors
  → Check network tab: Verify requests are successful

Search not working?
  → Type slowly - debounce is 300ms
  → Check there's data in history tab
  → Try refreshing page (Ctrl+R)

Progress bar stuck?
  → Check API returns status: curl http://localhost:9000/status/{id}
  → Verify polling is enabled: Check browser console logs
  → Try clicking monitor button for manual update

Styling issues?
  → Clear cache (Ctrl+Shift+R)
  → Check styles.css loads (Network tab in DevTools)
  → Try different browser
  → Check console for CSS errors (F12)

================================================================================
DEPLOYMENT
================================================================================

DEVELOPMENT:
  python skill_bridge.py

PRODUCTION (with Gunicorn):
  pip install gunicorn
  gunicorn -w 4 -b 0.0.0.0:9000 skill_bridge:app

DOCKER:
  FROM python:3.9
  WORKDIR /app
  COPY . .
  RUN pip install -r requirements.txt
  EXPOSE 9000
  CMD ["python", "skill_bridge.py"]

SYSTEMD SERVICE:
  Create /etc/systemd/system/skill-bridge.service
  See DASHBOARD.md for full configuration
